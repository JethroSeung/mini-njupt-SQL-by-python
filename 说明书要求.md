# 《数据库管理系统原型的软件说明书》撰写要求汇总
本报告即课程要求的实验报告/软件说明书，需覆盖三次实验的对应实现内容，基于 mini_base_blank 框架的开发成果撰写，是最终作业评分的核心依据之一。

## 一、各实验对应撰写内容
需按实验模块分章节撰写对应开发内容，覆盖三次实验的全部实现细节：

### 实验1：DBMS 存储实现
撰写存储模块的实现说明，覆盖表模式存储、表实例文件设计，以及 `schema_db.py`、`storage_db.py`、`head_db.py`（若实现缓存）等模块的开发内容。

### 实验2：DBMS 的 SQL 语句解析
撰写 SQL 解析与执行模块的实现说明，覆盖词法分析、语法分析、逻辑查询计划生成与执行等模块（`lex_db.py`、`parser_db.py`、`query_plan_db.py`、`node_db.py` 等）的开发内容。

### 实验3：事务持久性实现
1. 清晰说明日志机制的**关键数据结构**与**技术实现步骤**，包含前像文件、后像文件、活动事务表、提交事务表的设计，以及“提交规则”“先记后写规则”的落地逻辑。
2. 如有条件，补充事务持久性功能的实测结果与验证说明。

## 二、说明书格式与内容规范
### （一）基础要求
- 文档格式：Word 文档
- 篇幅要求：至少 3 页
- 必备组成：封面、目录、程序整体架构图、新增与修改模块描述

### （二）封面规范
封面必须标注以下信息：
- 班级、组号
- 每位组员的学号与姓名
- 每位组员对应的组内评分加减分模式
- 规则说明：若分组 Excel 中报备的评分模式与本说明书标注不一致，以软件说明书内容为准。

### （三）目录规范
建议基于文档标题自动生成目录。

### （四）程序架构图要求
1. 绘制完整的程序架构图，说明各模块的功能与相互调用关系。
2. 重点标注并说明程序的主入口函数。

### （五）新增/修改模块说明
仅列出在 mini_base_blank 框架基础上**新增或修改**的模块，未改动的模块无需列出。每个模块需包含以下内容：
1. 新增/修改内容所属的模块文件名称
2. 新增/修改的函数、类的代码位置与说明：
   - 函数：明确输入参数、输出结果、核心功能
   - 类：说明类的作用、各成员变量的作用
3. 关键算法：完整描述算法执行过程
4. 实测验证：列出实测数据与运行截图

## 三、配套代码规范（与报告评分直接关联）
代码规范是评分核心维度之一，需与说明书描述一致：
1. 所有新增函数必须添加注释，明确说明输入、输出与功能，对关键步骤和核心算法添加注释说明。
2. 修改的代码文件，需在文件开头注释作者的英文姓名与联系邮箱，便于代码维护。
3. 新增或修改的函数，需通过注释标注对应作者。
4. 确保程序可直接从 `main` 函数启动运行，无需额外配置调整。


## 四、评分参考维度
教师结合软件说明书与程序代码评定 A/B/C/D 四个等级（对应优良中差），评分参考维度如下：
- 程序整体功能涵盖度
- 函数功能完整性与正确性
- 注释规范度（输入、输出、执行步骤注释）
- 出错处理能力
- 结果展示清晰度
- 性能分析完整性
